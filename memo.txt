MPA (Multi Page Application) 의 단점

- 페이지 이동이 매끄럽지 않고 비 효율적임, 다수의 사용자 접속시, 서버의 부하가 심해 짐

npm i react-router-dom

// 1. "/" : 모든 일기를 조회하는 Home 페이지
// 2. "/new" : 새로운 일기를 작성하는 New 페이지
// 3. "/diary" : 일기를 상세히 조회하는 Diary 페이지

라우트 페이지 경로에서 없는 페이지 즉 404 같은 페이지는 경로를 path에 \* (와일드 카드) 경로로 지정
Routes 안에는 Route만

동적 경로 -> 동적인 데이터를 포함하고 있는 경로
- URL Parameter  ->  / 뒤에 아이템의 id를 명시
- Query String  ->  ? 뒤에 변수명과 값 명시  ( ~/search?q= 검색어 )

Query String 사용법은 useSearchParams 를 import 하고 useState처럼 사용
밑에는 예시
const [params, setParams] = useSearchParams();
console.log(params.get("value"))   

동적 페이지 이동은 useNavigate 사용으로 Link로 연결

### 이미지를 assets 경로에 넣은건 최적화를 위해서 (DATA URL로 캐싱되어 매번 불러오지 않음)
이미지가 엄청 많으면 경우가 또 다르니 그때는 public에 저장하는게 나음 소수의 이미지일때 assets
- public인 경우 -> <img src={"/emotion1.png"} />
- assets인 경우 -> import 로 불러온다음 <img src={emotion1} />
